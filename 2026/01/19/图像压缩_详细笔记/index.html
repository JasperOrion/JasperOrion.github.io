图像压缩 — 完整教学笔记（含代码）

《数字图像处理》教学笔记 · 第五章
Python 3.12 + NumPy + Matplotlib

一、概述

1.1 核心思想：三类冗余

冗余类型 定义 去除方法
编码冗余 灰度分布不均但等长编码 变长编码（哈夫曼/算术）
像素间冗余 相邻像素高度相关 预测编码(DPCM)/变换(DCT/DWT)
心理视觉冗余 人眼对高频不敏感 量化（丢弃高频系数）

1.2 通用压缩模型

1.3 分类

按重建 按原理 按码长
无损：哈夫曼/算术/LZW/RLE 统计编码 定长编码
有损：DCT(JPEG)/DWT(JPEG2000) 字典编码/预测编码/变换编码 变长编码

二、六大压缩编码技术（含代码）

2.1 哈夫曼编码

核心： 高频符号短码，低频符号长码 → 最优前缀码。

示例： A:0.4, B:0.3, C:0.2, D:0.1 → 码字 A=0, B=10, C=110, D=111，平均码长 1.9 bit vs 等长 2 bit。

特性： 前缀性（解码唯一）、即时性（收到即可解码）、最优性（最小平均码长）。

2.2 算术编码

核心： 将整个消息编码为一个 [0,1) 中的实数子区间，区间长度 = 消息联合概率。

示例： 编码 “BAC” (概率 A:0.6, B:0.3, C:0.1)：

解码 0.77 → B → A → C ✓

与哈夫曼的区别： 哈夫曼每符号独立编码为整数比特，算术编码整个消息为一个分数比特 → 更接近熵极限。

2.3 LZW 编码

核心： 动态构建”字符串→编号”字典。解码端同步重建字典，无需传输。

示例： 编码 “ABABABAB” → 输出 [65, 66, 256, 258, 66]（5个索引代表8个字符）。字典动态扩展：256=AB, 257=BA, 258=ABA。

解码关键： 每处理一个索引后，用 上一个输出串+当前输出串首字符 扩展字典，与编码端严格同步。

2.4 行程编码 (RLE)

核心： 连续相同值替换为 (值, 重复次数) 二元组。

图像类型 RLE 压缩比
二值图（文字扫描） 10:1 ~ 100:1
自然图像 可能 < 1:1

2.5 DPCM 预测编码

核心： 不编码像素值，编码预测值与实际值的差值（残差）——残差服从拉普拉斯分布，集中于零附近。

2D 预测器： $\hat{f}(x,y) = 0.5f(x-1,y) + 0.5f(x,y-1)$

2.6 小波变换编码 (DWT)

核心： DWT 将图像分解为 LL(低频逼近)/HL(水平边缘)/LH(垂直边缘)/HH(对角细节) 子带，能量集中在 LL。

DWT vs DCT：

特性 DCT (JPEG) DWT (JPEG2000)
基函数 余弦波（全局） 小波（局部）
块效应 有（8×8边界可见） 无
分辨率可伸缩 不支持 支持
低比特率性能 一般 更好

三、综合对比

六种算法总览

算法 类型 原理 典型CR 应用
哈夫曼 统计编码 高频短码 ~1.5:1 JPEG熵编码、PNG
算术编码 统计编码 区间收缩 ~1.5:1↑ H.264 CABAC
LZW 字典编码 动态字典 2~5:1 GIF、TIFF
RLE 游程编码 连续值压缩 >10:1(纯色) 传真、JPEG零系数
DPCM 预测编码 预测+残差 2~4:1 JPEG无损、H.264
DWT 变换编码 多分辨率+量化 10~50:1 JPEG2000

选择建议

  1. 纯色/简单图形 → RLE
  2. 已知概率分布 → 哈夫曼
  3. 概率动态变化 → 算术编码
  4. 自然图像无损 → DPCM + 哈夫曼
  5. 自然图像有损 → DWT (JPEG2000) / DCT (JPEG)

实际标准中的组合
