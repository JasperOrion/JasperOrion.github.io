线性表的链式存储

​ 与顺序表相比，链式存储仅仅只是在逻辑上是一个连续的线性表，各个结点在内存中的位置并不连续.。(这里引入线性表结点之间的关系:除第一个元素外,每个元素均有一个前驱;除最后一个元素外,每个元素均有一个后继).。顺序存储时，元素在一个连续的内存空间中，自然而然有自己的前驱和后继；而链式存储因其存储分散，就使用了指针来指向它的后继（当然这是指的单链表，后续的双链表、循环链表增加了指针指向来方便操作）。因此链式存储中插入和删除结点只需要修改指针即可(O(1)的时间复杂度),但查找和取值则退化到O(n)了

​ 这里简单介绍一下C++中的引用操作—— &，就是为该变量起一个别名，对别名进行修改时，原变量的值也会被修改。和指针的效果其实是差不多的，但不用进行解引用操作，更加易用。

  • 如何判断链表为空（初始化链表时链表的状态）
  • 如何判断结点 p 是否为表头/表尾（用于向前或向后遍历链表）
  • 如何在表头、表中和表尾插入/删除一个结点（处理多种情况）

一、单链表

1.结构定义

​ 要表示一个链表时只需要声明一个头指针L，用来指向单链表的第一个结点，所以LNode *L和LinkList L二者的效果其实是相同的，但是为了便于强调区分链表和结点，使用前者是更强调这是一个结点，后者强调这是一个链表。

2.初始化单链表&是否为空

​ 可以区分为两种，即是否带头结点——链表的第一个结点，通常不存储实际的内容。带头结点时，在进行操作首元结点时和其他结点的操作流程是相同的，毕竟首元结点也有了自己的前驱结点嘛；不带头结点时，在进行首插和首删时需要频繁改变头指针的指向，操作会显得复杂。

3.单链表的插入删除

（1）按位序插入

ListInsert (&L,i,e)在表L的第 i 个位置插入指定元素 e，需要找到第 i-1 个位置

假设现在链表长度为 n ，我们分别来分析插入位置分别为 <1，1，i(1<i<=n)，n+1，>n+1这几种情况。

  • 当 i < 1 || i >n+1 时，二者插入位置本身并不合法，处理时：前者直接返回false即可，后者需要判断待插入结点的前一个结点是否为NULL，循环结束后cur_p == NULL
  • i = 1 时前驱结点即为头结点，不会进入循环，直接进行插入
  • 1 < i <= n 时，循环结束后前驱结点分别为 1~n 。特别的，在 i = n 时，最后一个结点的next指向为NULL，这一不会影响我们的操作，此时新结点也将会指向NULL，成为尾结点。

显然可以看出我们在进行按位序插入结点时，需要先找到第 i 个结点，才能进行插入操作，总体来看其时间复杂度为O(n)的

不带头结点的链表头插时，需要更改头指针 L 的指向，将其指向新的结点。

（2）指定结点的后插操作

InsertNextNode (LNode *p, int e)在指定结点后插入元素 e

（3）指定结点的前插操作

（1）InsertPriorNode (LinkList &L, LNode *p, int e)在p结点前插入结点e

（2）InsertPriorNode ( LNode *p, int e)在p结点前插入结点e

​ 只知道一个结点现在只能访问其后的其他结点，（1）这时需要传入链表的头指针，然后查找 p 的前驱节点，之后在进行传统的操作了，显然这种需要O(n)的时间复杂度。（2）虽然无法直接将newNode插入到 p 结点前，但可以将其插入到后面，然后交换这两个结点中的数据域，以此实现O(1)的时间复杂度。

（4）按位序删除

ListDelete (LinkList &L, int i, int &e)删除表L中第 i 个结点，并用 e 返回值

依然需要寻找到第 i - 1 个结点，且它的next不能指向NULL
