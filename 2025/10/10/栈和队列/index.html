栈和队列

发表于2025-10-10|更新于2025-10-10|数据结构
|浏览量:

栈和队列二者可以看作一种特殊的线性表，缩减了他们的操作运算，让其只能在一端进行插入或删除操作。

一、栈

1.栈的基本操作

InitStrck(&S)：初始化空栈，构造一个空栈S并为其分配空间

DestroyStack(&L)：销毁栈，销毁栈并释放栈S所占用的内存空间

Push(&S,x)：进栈，若栈未满，，将元素x加入栈，让及成为新的栈顶

Pop(&S,&x)：出栈，若栈为空，将栈顶元素弹出，并用 x 返回

GetTop(s,&x)：读取栈顶元素，若栈非空，用 x 返回栈顶元素的值

StackEmpty(S)：判断栈是否为空

n个不同元素进栈，出栈元素不同排列顺序的个数为
$$
\frac{1}{n+1}C_{2n}^n
$$

二、队列

1.队列的基本操作

InitQueue(&Q)：初始化空栈，构造一个空栈S并为其分配空间

DestroyQueue(&Q)：销毁栈，销毁栈并释放栈S所占用的内存空间

EnQueue(&Q,x)：进栈，若栈未满，，将元素x加入栈，让及成为新的栈顶

DeQueue(&Q,&x)：出栈，若栈为空，将栈顶元素弹出，并用 x 返回

GetHead(Q,&x)：读取栈顶元素，若栈非空，用 x 返回栈顶元素的值

QueueEmpty(Q)：判断栈是否为空

2.顺序实现

（1）定义和初始化

​ 队尾指针 rear 指向接下来将要入队的位置，队首指针 head 指向接下来将要出队的位置，当 head 指向的位置与 rear 指向的位置相同，说明没有要出队的元素了，即队列为空。

（2）入队，出队，读队首

​ 三个操作需要进行判断队列是否为空或已满。为空时上文也提到过Q.front = Q.rear = 0;，与栈相比，栈对元素的操作只有一端，判断栈空栈满的条件只需要判断top的位置即可；而队列则是操作两端，用两个指针判断空或满，随着出队操作的进行，front前方的空间会空余出来，这时就要用到取余操作——Q.rear = (Q.rear + 1) % MaxSize，Q.front = (Q.front + 1) % MaxSize ，形成一个环形的结构。

（3）队列为空/满

  • 队列为空时上文已提到过（参见初始化队列）
  • front：指向将要出队元素的位置
  • rear：指向将要插入元素的空位置
  • 队列的元素个数：length = (rear + MaxSize - front) % MaxSize

如果让整个空间全部插满元素，rear理应和front指向同一个元素，这会导致队列满和队列空的条件相同😭，这是个问题，下面介绍几种方法。

  1. 直接避免全部插满的情况，单独空余出一个空间用于判断，当rear指向最后一个空余位置，也就是 front 的前一个位置(Q.rear + 1) % MaxSize,队列为空
  2. 既然要区分队空和队满，何不直接在定义时直接声明一个size变量用于记录队列元素的个数，size ==0 时队列为空；size == MaxSize 时队列为满，而且还能用于统计元素的个数。
  3. 可以设想：队满前的操作必定是入队操作；队空前的操作必定是出队操作或是刚初始化队列，我们可以给队列一个初始时为false的标签 bool tag = false,用于判断rear == front时前一个操作是否为入队。队列已满：front == rear && tag ==true。
