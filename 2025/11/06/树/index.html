树

一.树的基本概念

1.基本术语

  • 结点间的关系描述：祖先，子孙，双亲，孩子，兄弟，堂兄弟，路径（从上往下），路径长度
  • 结点、树的属性描述 ：结点层次，高度，结点的度；树的高度，树的度（结点度的最大值）
  • 有序树、无序树：树中各个结点的各个子树从左至右有无次序，时都能互换
  • 森林：m（m>=0）棵互不相交的树的集合

2.基本性质

  • 结点数 = 总度数 + 1
  • 度为m的树一定是非空树，至少有m+1个结点
  • m叉树的第i层至多有 **m^i-1^**个结点
  • 高度为h的m叉树至多有**(m^h^-1)/(m-1)个结点，至少有h**个结点.
  • 高度为h，度为m的树有h+m-1个结点
  • 具有n个结点的m叉树的最小高度为[logm(n(m-1)+1)]上取整
    • (m^h-1^-1)/(m-1) < n < (m^h^-1)/(m-1) : 取log

3.各种树

  • 二叉树
  • 满二叉树
  • 完全二叉树
  • 线索二叉树
  • 二叉搜索树
  • 二叉排序树 ：对于结点值有： 左孩子 < 双亲 < 右孩子
  • 平衡二叉树 : 树上任意结点的左子树和右子树的深度只差不超过 1.
  • 平衡二叉搜索树
  • 红黑树
  • B树
  • B+树
  • 哈夫曼树

二.存储结构

1.双亲表示法

结点结构
data | parent

​ 通常使用数组进行实现，空间效率高(不用额外管理内存开销)，实现简单，查找祖先的效率更高,给定一个连续的存储空间,用于存放树的各个结点,结点中可以仅仅存放结点的数据,以及双亲结点在数组中的下标,当作指针向根的方向访问,因根节点并没有双亲,令它的 parent = -1 .

​ 这样从孩子节点向上查找父节点，根节点就非常方便，但是如果想要获得兄弟结点的信息就只能遍历整个数组进行查找

2.孩子表示法

孩子结点 child_index | *next

表头结构 data | *first_child

树结构 数组 | 根位置 结点数目

​ 遍历时有些像层序遍历,不过这是在遍历每个结点和其孩子结点

3.孩子兄弟表示法

结点结构 data | first_child | right_sib

​ 很明显这种方式可以将一棵树转化成一棵二叉树,结点指向子结点和兄弟结点,方便查找某个结点的某个孩子,也可以添加一个parent指针域指向父结点

二叉树

一. 二叉树性质

  • 第i层最多2^i-1^个结点
  • 深度为h的树最多有2^h^-1个结点,此时该树为满二叉树
  • 对任意一棵二叉树,设度为2的结点数为 n2 ,度为1的结点数为 n1 度为0的结点(叶子结点)数为 n0 ,结点总数 **n **,度数之和为 d,则 n2+1 = n0.证明如下:
    • 若完全二叉树有2k个结点,则必有 n1 = 1 ,n0 = k , n2 = k-1
    • 若完全二叉树有2k-1个结点,则必有 n1 = 0 ,n0 = k , n2 = k-1

n = d + 1

n = n2 + n1 + n0

n = 2n2 + n1 + 1 = d + 1

  • 具有n个结点的完全二叉树深度为 k = [log2n]+1,证明如下:

2^k-1^-1< n <=2^k^-1 ==== >> 2^k-1^<= n <2^k^

k-1 <= log2n < k ==== >> k = [log2n]+1

  • 一颗结点数 n 的完全二叉树,结点按层序从左到右排号,1~n,对于任意结点i
    • 若 i = 1, 该节点为根节点,
    • 若 i > 1, 该节点的双亲结点是 [i/2]
    • 若 2i > n, 该节点无左孩子,否则左孩子为 2i
    • 若 2i+1 > n, 该节点无右孩子,否则右孩子为 2i+1
  • 完全二叉树，结点i<=n/2时为分支结点，否则为叶子节点

二. 存储结构

1. 顺序存储

​ 用一维数组存储二叉树的结点，结点的位置(在数组中的下标)要满足二叉树的结点间的逻辑关系(上面几条),由此可以看出,普通二叉树会在该存储结构中存在空位,斜树尤为如此,但完全二叉树可以占满,常用于完全二叉树

2.链式存储,二叉链表

​ 针对顺序结构所受的限制,采用二叉链表可以解决浪费内存的缺点,为其设置一个数据域两个指针域即可.也可继续添加一个指针域指向其父结点

​ *left_child | data | *right_child

​ *left_child | data | *right_child | *parent

三. 遍历方法

1.前 中 后 序遍历

  • 递归进行遍历,每个节点和其孩子节点都可当作一个简单的二叉树
  • 假如把二叉树的遍历顺序,用一个指针来标记,那指针移动的顺序(图三)
  • 遍历的顺序是相同的,也就是经过每个结点的顺序是相同的
  • 实际上,对于每个结点来说,均访问了三次,这三次的顺序就是前,中,后的时机
    • 第一次:经过该节点
    • 第二次:从左子结点返回到该节点
    • 第三次:从右子结点返回到该结点
